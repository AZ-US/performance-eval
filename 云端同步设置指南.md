# 绩效考核云端同步设置指南

目标：总经理打开专用链接，填完员工考评后点击“保存并提交云端”；Amy 在任意电脑打开同一个专用链接，点击“同步云端记录”即可看到提交结果。

## 1. 创建 Google Sheet

1. 打开 Google Drive，新建一个 Google Sheet。
2. 文件名建议：`绩效考核提交记录库`。
3. 打开菜单：`扩展程序` -> `Apps Script`。

## 2. 粘贴后端代码

1. 删除 Apps Script 默认代码。
2. 粘贴本项目的 [google-apps-script-backend.js](/Users/amyzhou/绩效考核/google-apps-script-backend.js) 全部内容。
3. 点击保存。

## 3. 发布 Web App

1. 点击 `部署` -> `新建部署`。
2. 类型选择：`Web 应用`。
3. `执行身份` 选择：`我`。
4. `谁可以访问` 选择：`知道链接的任何人`。
5. 点击部署，并授权。
6. 复制生成的 Web App URL，格式通常类似：

```text
https://script.google.com/macros/s/xxxxxxxxxxxx/exec
```

## 4. 生成总经理专用链接

1. 打开绩效考核页面：

```text
https://az-us.github.io/performance-eval/
```

2. 把 Web App URL 粘贴到“云端提交地址”。
3. 点击“保存地址”。
4. 页面顶部会自动生成一个带 `?cloud=` 的总经理专用链接。
5. 点击“复制链接”，把这个专用链接发给总经理。

## 5. 日常使用

总经理：

1. 打开专用链接。
2. 填写员工姓名、月度奖金、必要备注。
3. 每完成一个员工，点击“保存并提交云端”。

Amy：

1. 打开同一个专用链接。
2. 点击“同步云端记录”。
3. 在“员工记录库”里查看员工横向比较和个人纵向追踪。

## 注意

- 如果总经理用普通链接打开，没有带 `?cloud=`，他需要手动填写云端地址；所以建议只发“专用链接”。
- 如果网络或 Google 服务异常，仍可以点击“导出提交包”作为兜底。
- 这个专用链接不需要经常更新，除非你重新部署了 Apps Script 并生成了新的 Web App URL。
