# 绩效考核云端同步设置指南

目标：总经理打开链接，填完员工考评后点击“保存并提交云端”；Amy 在任意电脑打开同一个链接，点击“同步云端记录”即可看到提交结果。

当前线上版本已经内置云端提交地址。正常情况下，直接使用公开链接即可：

```text
https://az-us.github.io/performance-eval/
```

## 1. 创建 Google Sheet

1. 打开 Google Drive，新建一个 Google Sheet。
2. 文件名建议：`绩效考核提交记录库`。
3. 打开菜单：`扩展程序` -> `Apps Script`。

## 2. 粘贴后端代码

1. 删除 Apps Script 默认代码。
2. 粘贴本项目的 [google-apps-script-backend.js](/Users/amyzhou/绩效考核/google-apps-script-backend.js) 全部内容。
3. 点击保存。

## 3. 发布 Web App

1. 点击 `部署` -> `新建部署`。
2. 类型选择：`Web 应用`。
3. `执行身份` 选择：`我`。
4. `谁可以访问` 必须选择：`知道链接的任何人`。
5. 点击部署，并授权。
6. 复制生成的 Web App URL，格式通常类似：

```text
https://script.google.com/macros/s/xxxxxxxxxxxx/exec
```

如果打开 `/exec` 链接时跳到 Google 登录页，说明访问权限还没有设成“知道链接的任何人”。这种情况下总经理提交不会成功，需要回到 Apps Script 的部署设置里修改访问权限。

## 4. 生成总经理专用链接

如果以后更换了 Apps Script 部署地址，再按下面步骤生成新的专用链接。

1. 打开绩效考核页面：

```text
https://az-us.github.io/performance-eval/
```

2. 把新的 Web App URL 粘贴到“云端提交地址”。
3. 点击“保存地址”。
4. 页面顶部会自动生成一个带 `?cloud=` 的总经理专用链接。
5. 点击“复制链接”，把这个专用链接发给总经理。

## 5. 日常使用

总经理：

1. 打开公开链接或专用链接。
2. 填写员工姓名、月度奖金、必要备注。
3. 每完成一个员工，点击“保存并提交云端”。

Amy：

1. 打开同一个公开链接或专用链接。
2. 点击“同步云端记录”。
3. 在“员工记录库”里查看员工横向比较和个人纵向追踪。

## 注意

- 当前版本已经内置云端地址，所以普通公开链接也能提交。
- 如果以后更换 Apps Script 地址，建议重新复制带 `?cloud=` 的专用链接。
- 如果网络或 Google 服务异常，仍可以点击“导出提交包”作为兜底。
- 这个专用链接不需要经常更新，除非你重新部署了 Apps Script 并生成了新的 Web App URL。
